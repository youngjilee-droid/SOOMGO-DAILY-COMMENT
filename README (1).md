# 숨고 SA 성과 코멘트 생성기

숨고 SA 통합 리포트(`.xlsb`)를 업로드하면, **네이버/구글 x 요청/고수** 4개 매체-유형 조합에 대한
일간 성과 코멘트 초안을 자동으로 생성하는 Streamlit 앱입니다.

## 주요 기능

- 4개 시트(`일_네이버_요청`, `일_네이버_고수`, `일_구글_요청`, `일_구글_고수`)를 각각 파싱
- 매체·유형별 상단 탑라인 자동 생성 (광고비, REQ, CPR, 전주 동요일 대비 증감)
- 카테고리2 단위로 변동폭(REQ 절대 증감 기준)이 큰 순서대로 자동 선정 후 성과 요약
- 카테고리2 하위 서비스명 단위 상세 코멘트 (ㄴ 서브 불릿)
- 카테고리1 대비 해당 카테고리2가 역행/동행하는지 자동 코멘트
- REQ 변동률이 큰 서비스에는 `*키워드 성과 확인 필요` 문구 자동 삽입
  (현재 통합 리포트에는 키워드 레벨 데이터가 없어 표시만 하며, 추후 네이버 검색광고 API 연동 시
  실제 키워드 성과로 대체할 예정)
- 매체-유형별 개별 다운로드 + 4종 통합 다운로드 (.txt)

## 실행 방법

```bash
pip install -r requirements.txt
streamlit run app.py
```

브라우저에서 열리는 화면 좌측 사이드바에서:

1. 숨고 SA 통합 리포트(`.xlsb`) 업로드
2. 기준일(= '전일' 데이터 날짜) 선택
3. 필요 시 카테고리2/서비스 노출 개수, 최소 REQ 볼륨, 키워드 확인 플래그 임계값 조정
4. **코멘트 생성** 버튼 클릭

## 데이터 구조 전제

이 앱은 숨고 SA 통합 리포트의 다음 시트 구조를 전제로 합니다.

- 헤더는 5번째 행(0-indexed 기준 4번째 행)에 위치
- 컬럼 순서: `cate1`, `cate2`, `서비스`, `CAP`, 이후 **전일 → 전주 동요일 → gap → gap%** 순으로
  각 12개 지표(`노출수, 클릭수, 광고비, CTR, CPC, AB_UA(요청), AB_UA(고수), AB_REQ, AB_CAC, CPR, 도달률, CVR`)가 반복
- 데이터는 서비스(최하위 레벨) 단위 1 row로 구성

리포트 양식이 변경되면 `app.py` 상단의 `METRIC_NAMES`, `BLOCK_NAMES`, `HEADER_ROW_IDX`,
`DATA_START_ROW_IDX` 값을 리포트 구조에 맞게 수정해주세요.
